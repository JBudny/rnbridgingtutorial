#### Exposing Swift only library to ObjC

Sometimes you may have to use some Swift-only libraries and sooner or later, you'll have to use them in ObjC code.

In our case, we need to somehow use our datepicker library in ObjC module-implementation class.
To do that we will create small Swift wrapper class that will expose functionality we need.

```swift title="ios/FastisWrapper.swift"
import Fastis
import Foundation

@objc(FastisWrapper)
public class FastisWrapper : NSObject {
    @objc public static func present(
        withTitle title: String,
        viewController: UIViewController,
        onCancel: @escaping () -> Void,
        onResult: @escaping (Dictionary<String, Dictionary<String, Int>>) -> Void
    ) {
        let fastisController = FastisController(mode: .range)
        fastisController.title = title
        fastisController.doneHandler = { selectedRange in
            guard let selectedRange = selectedRange else {
                onCancel()
                return
            }
            let fromDateComponents = Calendar.current.dateComponents([.day, .month, .year], from: selectedRange.fromDate)
            let toDateComponents = Calendar.current.dateComponents([.day, .month, .year], from: selectedRange.toDate)
            onResult([
                "from": [
                    "day": fromDateComponents.day!,
                    "month": fromDateComponents.month!,
                    "year": fromDateComponents.year!
                ],
                "to": [
                    "day": toDateComponents.day!,
                    "month": toDateComponents.month!,
                    "year": toDateComponents.year!
                ]
            ])
        }
        fastisController.dismissHandler = { onCancel() }
        fastisController.present(above: viewController)
    } 
}
```

What's happening here?

- we are importing `Fastis` library
- we declare public `FastisWrapper` class that extend `NSObject` and is exported to ObjC (via `@objc(<exported-objc-name>)`)
- we create public static function `presentWithTitle:viewController:onCancel:onResult:` and export it to @objc

Because our exported function is static, we will not use [Delegate Pattern](https://developer.apple.com/documentation/swift/using-delegates-to-customize-object-behavior), but instead the function will accept two callbacks that will be invoked when the user picks the range of dates.

In the body of the function we initialize modal view controller with the datepicker and configure its properties.

Finally, modal view controller is presented above currently displayed one.
